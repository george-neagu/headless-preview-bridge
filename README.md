## Preview Login Flow

  The preview flow lets WordPress open unpublished posts in the React frontend.

  ### How it works

  1. WordPress creates a preview URL that points to the frontend:

     ```txt
     /api/preview?id=POST_ID&token=PREVIEW_TOKEN&status=draft

  2. The React app detects /api/preview and reads the id, token, and optional status query params.
  3. The frontend calls this local API route:

     /api/wp-preview?id=POST_ID&token=PREVIEW_TOKEN

  4. In development, Vite proxies that request to WordPress:

     /wp-json/headless-preview-bridge/v1/preview

  5. The plugin validates the token and returns the preview post content.

  ### Required URL parameters

  - id: the WordPress post ID.
  - token: the preview token generated by the plugin.
  - status: optional, used only for display in the frontend.

  Example:

  http://localhost:5174/api/preview?id=123&token=abc123&status=draft

  ### Frontend expectation

  The frontend expects the preview endpoint to return a post object shaped like the WordPress REST API post response, including:

  {
    "id": 123,
    "date": "2026-05-16T10:00:00",
    "title": {
      "rendered": "Post title"
    },
    "content": {
      "rendered": "<p>Preview content</p>"
    }
  }

  ### Notes

  The token should be treated as temporary access to preview content. Do not expose permanent credentials or require the user to be logged
  into WordPress from the frontend.
